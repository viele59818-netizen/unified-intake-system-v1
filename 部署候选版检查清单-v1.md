# 部署候选版检查清单 v1

## 目标

把当前原型收口成一个“可部署候选版”，用于后续挂到 `https://` 地址上做真实测试。

## 当前版本定位

- 当前版本：`v1-candidate`
- 当前形态：单服务前后端一体化原型
- 当前用途：本地验证通过，准备进入线上测试阶段

## 已具备能力

- 输入提交
- 图片/视频上传
- 专题聚合
- 状态切换
- 重点标记
- 专题备注
- 专题动作清单
- 专题摘要导出
- 动作清单导出
- 任务格式导出
- 专题删除

## 部署前必须确认

### 1. 功能收口

- 暂停继续增加新功能
- 当前功能先以“够用、可测、可演示”为目标
- 后续新增需求进入下一轮迭代

### 2. 数据策略

- 当前数据存在本地 `data/` 目录
- 部署测试环境时要接受“重启可能丢数据”这一点
- 如果需要保留测试数据，后续要补持久化策略

### 3. 访问方式

- 本地访问：`http://localhost:3001`
- 线上访问目标：`https://...`
- 微信中使用必须走 `https://`

### 4. 测试重点

- 页面是否能打开
- 提交是否成功
- 图片/视频是否能上传
- 专题功能是否正常
- 导出是否正常
- 手机端体验是否可接受

## 上线前建议手测项目

### 提交链路

- 新增一条输入
- 带图片提交
- 带视频提交
- 来源切换
- 专题自动聚合

### 专题链路

- 点击专题卡片
- 修改专题备注
- 添加动作
- 设置负责人和截止时间
- 标记动作完成
- 删除专题

### 输出链路

- 导出专题摘要
- 导出动作清单
- 导出任务格式

### 手机体验

- 手机端打开页面
- 表单是否好点
- 上传是否顺手
- 专题详情是否可读

## 当前已知限制

- 还不是微信原生入口
- 还没有正式登录/鉴权
- 还没有接飞书/企微正式入口
- 数据层仍是原型级实现

## 候选版上线策略

建议按这条顺序走：

1. 本地确认当前版本稳定
2. 部署成一个测试 `https` 地址
3. 用手机和微信打开测试
4. 收一轮真实使用反馈
5. 再决定是否进入下一版

## 当前结论

这版已经不需要继续横向加功能了，更值得进入：

- 部署
- 真实测试
- 反馈驱动迭代
