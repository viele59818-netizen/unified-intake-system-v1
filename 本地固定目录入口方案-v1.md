# 本地固定目录入口方案 v1

## 目标

把“微信等外部来源的原始材料”统一落到一个固定本地目录中，作为你和 AI 的共同入口。

这套方案解决的是：

- 微信不方便直接自动接入
- 飞书不适合做微信原始入口
- 但你已经验证过“保存到本地后我可以直接处理”

所以第一版最稳的入口，就是本地固定目录。

## 一句话方案

`微信/网页/飞书/邮件/截图/视频 -> 本地固定目录 -> 我读取处理 -> 输出到 Obsidian/飞书/滴答`

## 为什么这条路最适合现在

- 你已经实际用过，能跑通
- 不依赖微信开放接口
- 不依赖飞书权限先完全打通
- 图片、视频、截图、文档都能收
- 后续也容易扩成自动化

## 推荐目录位置

建议在一个长期固定的位置建一个入口目录。

推荐位置：

- `/Users/ethanlam/Documents/AI输入池`

如果你更喜欢在桌面，也可以：

- `/Users/ethanlam/Desktop/AI输入池`

我更推荐 `Documents`，因为更稳定，不容易太乱。

## 推荐目录结构

第一版先不要太复杂，建议这样：

```text
AI输入池/
  00_待处理/
  01_处理中/
  02_已归档/
  03_长期资料/
```

### 含义

- `00_待处理`
  - 新进来的内容先丢这里
- `01_处理中`
  - 正在分析的专题放这里
- `02_已归档`
  - 已经做完分析的专题放这里
- `03_长期资料`
  - 竞品资料、页面资料、通用参考资料等长期保留内容

## 专题目录结构

当你有一个具体专题时，在 `00_待处理` 或 `01_处理中` 下建一个专题文件夹。

建议命名：

```text
2026-04-22_产品舒适性问题/
```

专题目录内建议这样放：

```text
2026-04-22_产品舒适性问题/
  01_聊天记录/
  02_截图/
  03_视频/
  04_网页资料/
  05_补充说明/
```

## 每种资料怎么放

### 微信聊天

如果是文字：

- 复制到一个 txt 或 md 文件里
- 放到 `01_聊天记录`

如果是聊天截图：

- 直接放到 `02_截图`

### 图片和截图

- 直接放到 `02_截图`

### 视频

- 直接放到 `03_视频`

### 网页和链接

建议两种方式任选其一：

1. 建一个 `links.md`
2. 保存网页截图或导出 PDF 到 `04_网页资料`

### 补充说明

如果你想额外补一句背景，放到：

- `05_补充说明/说明.md`

## 你后面的实际动作

以后遇到事情时，只需要做这些最轻动作：

### 情况 1：微信里一段重要聊天

- 截图，放进专题目录的 `02_截图`
- 或复制文字，放进 `01_聊天记录`

### 情况 2：客户发了视频

- 保存视频
- 放进 `03_视频`

### 情况 3：一个网页或页面链接

- 放到 `04_网页资料/links.md`
- 或截图保存

### 情况 4：你想补一句背景

- 写进 `05_补充说明/说明.md`

## links.md 推荐格式

```md
# 链接资料

- 标题：
  - 链接：
  - 说明：

- 标题：
  - 链接：
  - 说明：
```

## 说明.md 推荐格式

```md
# 补充说明

- 这个专题是关于什么：
- 为什么现在要分析：
- 当前最关心的问题：
- 这批资料里哪个最重要：
```

## 推荐命名规则

专题目录建议固定为：

```text
日期_主题
```

例如：

- `2026-04-22_产品舒适性问题`
- `2026-04-25_竞品页面分析`
- `2026-04-28_客户A退货原因整理`

这样后面最好找。

## 后面可升级的方向

第一版先手动，但后面很容易升级：

### 升级 1：自动整理

- 新文件进 `00_待处理` 后自动提醒

### 升级 2：自动建专题目录

- 输入主题后自动生成结构

### 升级 3：自动转飞书

- 分析完成后自动生成飞书文档草稿

### 升级 4：自动回写任务系统

- 从分析结果里抽取动作，进入滴答清单
