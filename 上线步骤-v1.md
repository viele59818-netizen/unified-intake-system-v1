# 上线步骤 v1

## 当前前提

- 当前目录还不是 `git` 仓库
- 代码已经达到 `v1-candidate`
- 本地原型已验证可运行

## 最短上线路径

### 1. 建一个 Git 仓库

推荐把整个目录作为一个仓库：

```text
2026-04-22-new-chat/
```

这样可以同时保留：

- 前端原型
- 后端原型
- 部署配置
- 方法文档

### 2. 先跑预检脚本

```bash
zsh /Users/ethanlam/Documents/Codex/2026-04-22-new-chat/predeploy-check.sh
```

### 3. 推到 GitHub

推上去后，确认仓库根目录包含：

- `render.yaml`
- `intake-backend-v1/`
- `weixin-intake-v1/`

### 4. 在 Render 新建服务

方式：

- 直接导入 GitHub 仓库
- 让 Render 读取根目录 `render.yaml`

### 5. 拿到一个 `https://` 地址

例如：

```text
https://your-intake-service.onrender.com
```

### 6. 用微信和手机测试

重点测试：

- 页面能打开
- 新建输入
- 选图片
- 选视频
- 专题聚焦
- 导出是否正常

## 当前不建议直接做的事

- 不建议先接复杂鉴权
- 不建议先接微信原生入口
- 不建议先改成小程序

先把网页候选版上线测试，收益最高。

## 当前结论

下一步真正需要你做的动作只有一个：

`把当前目录变成一个 git 仓库并推上去`

后面部署就顺了。
