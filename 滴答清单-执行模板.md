# 滴答清单执行模板

## 任务标题命名

建议格式：

`[产品问题] {{问题主题}} - {{动作}}`

示例：

- `[产品问题] 舒适性问题 - 验证第二版贴合性`
- `[产品问题] 舒适性问题 - 补充客户版飞书文档`
- `[产品问题] 舒适性问题 - 跟进加热功能测试`

## 任务描述模板

```md
【问题主题】
{{问题主题}}

【任务目标】
{{这次要完成什么}}

【背景摘要】
{{一句话说明为什么要做}}

【输入资料】
- Obsidian 主笔记：{{链接或路径}}
- 飞书文档：{{链接}}

【完成标准】
- [ ] 标准 1
- [ ] 标准 2
- [ ] 标准 3

【输出要求】
{{最后要产出什么}}

【完成后回写】
- 更新飞书：是 / 否
- 更新 Obsidian：是 / 否
```

## 任务状态建议

- 待处理
- 进行中
- 待确认
- 已完成
- 已取消

## 任务优先级建议

- P1：直接影响客户、退货、上线决策
- P2：影响内部判断和协作推进
- P3：优化项、补充项、沉淀项

## 常见任务类型

### 验证类

- 验证第二版是否已解决某问题
- 验证不同体型适配情况
- 验证加热和力度表现

### 文档类

- 整理客户版文档
- 更新飞书结论
- 补充截图和视频说明

### 协调类

- 跟进产品改进项
- 与客户确认问题边界
- 同步售后与产品团队

### 复盘类

- 回写 Obsidian 主笔记
- 更新方法论
- 补充可复用模板
